[테스트 단위 기능 구현 관련 정리]

#공통
[x] 다시 시작이 잘 되어야 한다.
[x] 만약에 예외가 발생(throw)되었을 때 그 부분부터 다시 입력이 잘 되어야 한다.

#로또
[] 당첨 번호와 로또 번호를 비교했을 때 match number가 예상된 출력 결과여야 한다.
[x] 로또 번호는 오름차순으로 정렬되어야 한다.

#검증

1. 로또 관련
   (공통)
   [x] 숫자는 비어있을 수 없습니다.
   [x] 숫자는 자연수여야 합니다.  
   [x] 숫자는 1부터 45까지여야 합니다.

(로또)
[x] 배열 안의 값은 6개여야 합니다.
[x] 배열 안의 값은 중복되어서는 안됩니다.
(보너스 번호)
[x] 보너스 번호는 당첨 로또 번호와 중복될 수 없습니다.

2. 구매 관련
   [x] 구매금액은 숫자여야 합니다.
   [x] 구입 금액은 비어있을 수 없습니다.
   [x] 구입 금액은 자연수여야 합니다.
   [x] 구입 금액은 1000원 단위로 입력해야 합니다.
   [x] 구입 금액이 너무 커서는 안 됩니다. (MAX_INTEGER 이상)

3. 기타
   [x] 재입력 시 y/n 만 입력이 가능합니다.

#구매
[x] 구입한 금액 만큼 로또가 발행된다.
[x] 수익률이 맞아야 한다. (소수점 첫째자리 반올림)
